# How to contribute to Prometheus

Welcome and thank you for investing your time in contributing to Prometheus!

## Report a bug

Create an issue, label it as bug.

## Feature requests

Create an issue, label it as enhancement. Give as much info as possible.

## Code changes

Fork the repo, make changes on fork, [create a PR on a fork](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request-from-a-fork). [Enable maintainer edits](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork)

If it's related to an issue, link to it.

### Best practices

Checklist:

- Does the code work?
- Comments
- Unit tests
- Update docs

### Pull request guidelines

Use PR template, include as much info as possible to make it easier for us to review.

Make sure the commits are descriptive.

Good examples

Bad examples

Squish commits before making the PR ready to review.

## Doc changes

MMS style guide, [GitHub style guide](https://docs.github.com/en/contributing/style-guide-and-content-model/style-guide)

Content changes we accept:

- Technical and grammatical corrections
- Typo fixes
- Expanded explanations of existing features, when the expansion has a compelling reason

Content we do not accept:

- Edits purely for tone, readability, or efficiency.

[Best practices from GitHub docs](https://docs.github.com/en/contributing/writing-for-github-docs/best-practices-for-github-docs)

## Questions?

Open a `questions` issue or contact the authors using the email addresses listed on Inspire-HEP.

We'll get to it as soon as we can, but it may not be quick.
